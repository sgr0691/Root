# Contributing to Root

## Issue-Only Policy

Root is currently **issue-only**. We welcome bug reports, feature requests, and
discussions, but we do **not** accept external pull requests.

### Why?

The modern open-source ecosystem faces an increasing wave of supply-chain attacks
targeting package managers and developer tooling. AI-aided code generation has
made it trivial to produce plausible-looking but malicious contributions at scale.

Until Root has a formal review and signing pipeline for third-party code, accepting
PRs would introduce unacceptable risk to users who trust Root with their machine
environment.

### What you can do

- **Report bugs** — Open an issue with clear reproduction steps
- **Request features** — Open an issue describing the use case
- **Discuss design** — Open an issue for architecture or design questions
- **Share agent skills** — Open an issue linking to your skill file

### What happens next

The maintainers review all issues regularly. Priority is given to:

1. Security and stability issues
2. macOS compatibility (Root's primary platform)
3. AI agent integration improvements
4. Documentation gaps

## Security Disclosures

If you find a security vulnerability, **do not open a public issue**. Instead,
reach out directly via the repository's security advisory system or contact
the maintainers through the GitHub Security tab.

## Code of Conduct

All interactions must follow the [Code of Conduct](CODE_OF_CONDUCT.md).
